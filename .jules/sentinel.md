## 2025-04-26 - [Arbitrary File Upload DoS in NestJS]
**Vulnerability:** The `@UseInterceptors(FileInterceptor(...))` was used on a route handler in `ConfigurationsController` with manual authorization checks (`if (req.user.role !== "ADMIN")`) performed *inside* the route handler body. This is vulnerable because NestJS Interceptors run *before* the route handler. Unauthenticated or non-admin users could upload files, which the `FileInterceptor` would save to disk, and only then would the route handler reject the request (leaving the file on disk), leading to a Denial of Service via storage exhaustion.
**Learning:** In NestJS, manual authorization checks inside the route handler do not protect `FileInterceptor` uploads. The interceptor executes first.
**Prevention:** To prevent this, authorization checks must be enforced using Guards (e.g., `@UseGuards(AuthGuard('jwt'), RolesGuard)` and `@Roles('ADMIN')`) directly on the route. Guards execute before Interceptors, correctly rejecting unauthorized requests before the file is parsed and saved to disk.

## 2024-05-01 - Absolute Path Traversal Bypass via path.join
**Vulnerability:** `files.service.ts` used `filename.includes("..")` to stop path traversal, but then used `path.join(cwd, uploads, filename)` to resolve paths. This allowed Absolute Path Injection: if an attacker supplied an absolute path (e.g., `/etc/passwd`), `path.join` interpreted it as a new root segment, completely bypassing the directory restriction while never triggering the `..` string check.
**Learning:** Checking for relative traversal sequences (`..`) is entirely insufficient when absolute paths can be passed directly to file system APIs like `path.join()`. Node's `path.join` natively resolves absolute path segments by snapping to root, masking the vulnerability.
**Prevention:** To secure path composition, always use `path.resolve(basePath, filename)` to compute the final path, and then strictly enforce boundaries by checking if the resolved path starts with the base path including the platform separator (`targetPath.startsWith(basePath + path.sep)`). Retain basic string checks if inputs are used across distributed authorization gates, but never rely on them exclusively.

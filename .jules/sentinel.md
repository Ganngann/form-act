## 2024-05-07 - Arbitrary File Upload DoS via Interceptor Priority
**Vulnerability:** In NestJS, `FileInterceptor` runs before the controller method executes. When authorization checks (like checking if a user is an ADMIN) were done inline within the controller method rather than using Guards, the interceptor would save the uploaded file to disk *before* the unauthorized user was rejected, creating an Arbitrary File Upload Denial of Service (DoS) vulnerability.
**Learning:** Framework execution order matters for security. Because interceptors execute before controller handlers, security boundaries (like AuthGuard and RolesGuard) must be evaluated at the Guard level to drop malicious requests before resource-intensive operations (like saving files to disk) occur.
**Prevention:** Always use `@UseGuards(RolesGuard)` and `@Roles('ADMIN')` on sensitive file upload endpoints rather than imperative `req.user.role !== 'ADMIN'` checks inside the controller logic.
